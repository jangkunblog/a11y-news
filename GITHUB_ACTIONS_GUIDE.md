# GitHub Actions 자동화 가이드

GitHub Actions를 사용하여 디지털 접근성 뉴스를 자동으로 수집하고 블로그에 게시하는 방법입니다.

## 🎯 Workflow 기능

### ⏰ 정기 자동 실행
- **시간**: 매주 월요일 오전 9시 (KST)
- **동작**: 자동으로 최신 접근성 뉴스 수집 → 마크다운 생성 → 커밋 → 푸시

### 🎮 수동 실행
- **위치**: GitHub 저장소 > Actions 탭
- **커스터마이징**:
  - 검색 키워드 입력 가능
  - Gemini에 전달할 추가 지시사항 입력 가능

## ⚙️ 설정 방법

### 1️⃣ Gemini API 키를 GitHub Secrets에 등록

1. GitHub 저장소 페이지로 이동
2. **Settings** 탭 클릭
3. 왼쪽 사이드바에서 **Secrets and variables** > **Actions** 클릭
4. **New repository secret** 버튼 클릭
5. Secret 추가:
   - **Name**: `GEMINI_API_KEY`
   - **Secret**: 발급받은 Gemini API 키 붙여넣기
6. **Add secret** 클릭

> ⚠️ **중요**: Secret 이름은 정확히 `GEMINI_API_KEY`로 입력해야 합니다!

### 2️⃣ Workflow 활성화

워크플로우 파일(`.github/workflows/accessibility_news.yml`)이 이미 저장소에 있으므로, 
코드를 GitHub에 푸시하면 자동으로 활성화됩니다.

```bash
git push origin main
```

## 🚀 사용 방법

### 자동 실행 (Schedule)

설정 완료 후 **매주 월요일 오전 9시(KST)**에 자동 실행됩니다.
별도 조치 불필요!

### 수동 실행 (Workflow Dispatch)

#### 기본 실행 (기본 키워드 사용)

1. GitHub 저장소 페이지 이동
2. **Actions** 탭 클릭
3. 왼쪽에서 **"Collect Accessibility News"** 선택
4. 오른쪽 **"Run workflow"** 버튼 클릭
5. Branch: `main` 선택
6. 입력 필드는 비워두고 **"Run workflow"** 클릭

#### 커스텀 키워드로 실행

1. Actions > Collect Accessibility News > Run workflow
2. **키워드** 입력 (쉼표로 구분):
   ```
   WCAG 3.0, ARIA 1.3, 웹 접근성 법규
   ```
3. Run workflow 클릭

#### 커스텀 프롬프트 추가

1. Actions > Collect Accessibility News > Run workflow
2. **추가 지시사항** 입력:
   ```
   특히 모바일 접근성에 초점을 맞춰 작성해주세요
   ```
3. Run workflow 클릭

## 📊 Workflow 단계

```
1. 저장소 체크아웃
   ↓
2. Python 환경 설정 (Python 3.11)
   ↓
3. 의존성 설치 (requirements.txt)
   ↓
4. 오늘 날짜 파일 존재 여부 확인
   ↓
5. Python 스크립트 실행
   - Google News RSS 검색
   - Gemini API로 요약 및 구조화
   - 마크다운 파일 생성
   ↓
6. 변경사항 확인
   ↓
7. Git 커밋 및 푸시 (변경사항 있을 경우)
   ↓
8. 실행 결과 요약 표시
```

## 🔍 실행 결과 확인

### Actions 탭에서 확인

1. **Actions** 탭 클릭
2. 최근 workflow run 클릭
3. **Summary** 섹션에서 결과 확인:
   - ✅ 상태: 성공/실패
   - 📄 파일: 생성된 파일 경로
   - 📅 실행 일시
   - 🎯 사용된 키워드 (있는 경우)

### 저장소에서 확인

`src/content/blog/` 폴더에 새 파일이 추가되었는지 확인

### 실제 블로그에서 확인

GitHub Pages가 설정되어 있다면:
- 약 1-2분 후 배포 완료
- `https://yourusername.github.io/kakao-a11y-news/blog`에서 확인

## 🎯 예시 시나리오

### 시나리오 1: WCAG 3.0 관련 뉴스만 수집

**수동 실행 시 입력:**
- 키워드: `WCAG 3.0, W3C Silver`
- 추가 지시사항: `WCAG 3.0의 변경사항에 집중하여 작성해주세요`

### 시나리오 2: 국내 접근성 법규 뉴스

**수동 실행 시 입력:**
- 키워드: `웹 접근성 인증, 한국 장애인차별금지법, 정보접근성`
- 추가 지시사항: `한국의 법규와 정책 변화를 중심으로 정리해주세요`

**수동 실행 시 입력:**
- 키워드: `ARIA live regions, screen reader compatibility, accessibility testing tools`
- 추가 지시사항: `개발자를 위한 실용적인 가이드 형식으로 작성해주세요`

## ⚡ 즉시 테스트 실행

설정 직후 바로 테스트하려면:

1. GitHub 저장소 > **Actions** 탭
2. **Collect Accessibility News** 선택
3. **Run workflow** 클릭
4. 기본값으로 **Run workflow** 실행
5. 실행 상태 모니터링 (약 1-3분 소요)
6. 완료 후 `src/content/blog/` 폴더 확인

## 🐛 문제 해결

### ❌ "GEMINI_API_KEY가 설정되지 않았습니다" 오류

**원인**: GitHub Secrets에 API 키가 없음

**해결**:
1. Settings > Secrets and variables > Actions
2. GEMINI_API_KEY Secret 추가

### ❌ Workflow가 실행되지 않음

**원인**: 저장소에 workflow 파일이 없음

**해결**:
```bash
git add .github/workflows/accessibility_news.yml
git commit -m "Add accessibility news workflow"
git push origin main
```

### ❌ "permission denied" 오류

**원인**: Workflow에 write 권한 없음

**해결**:
1. Settings > Actions > General
2. **Workflow permissions** 섹션
3. **Read and write permissions** 선택
4. Save

### ⚠️ "수집된 뉴스가 없습니다"

**원인**: 
- 키워드가 너무 구체적
- 해당 날짜에 관련 뉴스가 없음

**해결**:
- 더 일반적인 키워드 사용
- 다른 날짜에 재시도

### ⚠️ API 할당량 초과

**원인**: Gemini API 무료 티어 한도 초과

**해결**:
- 24시간 후 재시도
- 유료 플랜 고려
- 실행 빈도 조정 (주 1회 → 월 2회)

## 🔒 보안 주의사항

### ✅ 안전한 관리

- ✅ API 키는 GitHub Secrets에 암호화 저장
- ✅ Secret은 로그에 노출되지 않음
- ✅ Secret은 public fork에서 접근 불가

- ❌ API 키를 커밋 메시지에 포함
- ❌ `.env` 파일을 Git에 커밋

## 🎨 커스터마이징

### Cron 스케줄 변경

`.github/workflows/accessibility_news.yml`:

```yaml
schedule:
  - cron: '0 0 * * 1'  # 매주 월요일 오전 9시(KST)
  # - cron: '0 0 1,15 * *'  # 매월 1일, 15일
  # - cron: '0 0 * * *'  # 매일
```

### 타임존 참고

| 설명 | Cron 표현식 (UTC) | KST 시간 |
|------|-------------------|----------|
| 매주 월요일 오전 9시 | `0 0 * * 1` | 월 09:00 |
| 매일 오전 10시 | `0 1 * * *` | 매일 10:00 |
| 매월 1일 오전 9시 | `0 0 1 * *` | 1일 09:00 |

## 💡 고급 활용

### 다중 Workflow 설정

여러 주제별로 다른 workflow 생성:

- `accessibility_news.yml` - 일반 접근성 뉴스

각각 다른 스케줄과 키워드로 실행 가능!

### Slack/Discord 알림 추가

workflow에 알림 단계 추가:

```yaml
- name: Notify Slack
  if: steps.check_changes.outputs.has_changes == 'true'
  uses: slackapi/slack-github-action@v1
  with:
    webhook-url: ${{ secrets.SLACK_WEBHOOK_URL }}
    payload: |
      {
        "text": "새로운 접근성 뉴스 게시물이 생성되었습니다! 🎉"
      }
```
